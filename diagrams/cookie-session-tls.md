Title: Cookie / Session / TLS (Simple)

HTTP = browser sends requests to server (not encrypted)

HTTPS = HTTP + TLS (encrypted)

TLS protects:

confidentiality (no one reads)

integrity (no one edits)

server authentication (real website)

Cookie

stored in browser

sent with requests

often contains a session id

Session

stored on server

server-side memory of logged-in user

cookie points to the session
